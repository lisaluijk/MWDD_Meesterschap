                ## Peter-Paul Koch — browsers & support detection

                ### Wat is een browser?

                * Interpreteert HTML, CSS en JavaScript
                * Levert de DOM aan (Document Object Model = soort kopie van HTML die JS kan gebruiken)
                * Ook AOM = Accessibility Object Model, bedoeld voor screenreaders en andere assistive tech
                * Heeft natuurlijk ook een interface nodig
                * De engine zelf is officieel geen onderdeel van de browser

                ### Parser

                Parser = stukje software dat een string omzet naar instructies voor een programmeertaal.

                ### Layout / rendering

                Oppassen met layouts aanpassen (kolommen etc). Eerst goed bedenken wat je nodig hebt, anders wordt het snel chaos of onoverzichtelijk.

                Rendering engine = eigenlijk alles behalve de JS-engine en de interface.
                Verantwoordelijk voor:

                * HTML parsing
                * CSS parsing
                * rendering
                * opbouwen van DOM en AOM trees

                Bekende engine: Chromium/Blink.

                ### JavaScript / rendering blocking

                Core = wat JavaScript echt een programmeertaal maakt.

                Render blocking = browser wacht met renderen omdat eerst CSS of JS geladen moet worden.

                JavaScript liefst onderaan je HTML zetten, omdat document.write eerst klaar moet zijn.

                Daarom bestaan ook defer en async.

                * defer doe dit later
                * async = maakt niet uit wanneer, als het maar niet nu meteen hoeft

                CSS juist eerder laden dan HTML rendering.

                Slechte laadtijden van websites komen vaak gewoon door slecht geschreven code / verkeerde structuur.

                ---

                ## Backward compatibility

                Als iets ooit in een browser gewerkt heeft, moet het basically voor altijd ondersteund blijven.

                Zonder doctype gaat HTML soms in soort IE6-compatibility mode.

                ### Oude HTML dingen

                [form object].elements

             

                 = oude manier van partials/herbruikbare stukken zoals headers.
                iframe komt hier eigenlijk vandaan.

                    Er bestaat blijkbaar ook een lijst met historische CSS-fouten die browsers nog steeds moeten ondersteunen.

                    # Geschiedenis van browsers

                    ## WWW Browser (1990)

                    Uitgevonden door Tim Berners-Lee.

                    Eerste browser ooit.

                    In 2019 opnieuw gebouwd:
                    [WorldWideWeb browser rebuild](https://worldwideweb.cern.ch/?utm_source=chatgpt.com)

                    ---

                    ## IP / TCP / UDP

                    * IP = soort adressensysteem van internet, zoekt automatisch routes om problemen heen
                    * TCP = mét foutcontrole
                    * UDP = zonder foutcontrole → sneller, handig voor video/games maar kan haperen

                    ---

                    ## Mosaic (1993)

                    Eerste browser die afbeeldingen kon tonen op webpagina’s. Grote doorbraak.

                    ---

                    ## Netscape (1994)

                    Eerste “echte” commerciële browser.

                    Werd letterlijk op cd-roms naar mensen thuis gestuurd.

                    ---

                    ## Internet Explorer (1995)

                    Van [Microsoft](https://www.microsoft.com?utm_source=chatgpt.com).

                    Belangrijk omdat hij gratis was.

                    ---

                    ## Opera (1995)

                    Nu beetje sketchy/crypto-achtig, maar vroeger best belangrijk.

                    Een van de oprichters werkte ook mee aan CSS.

                    ---

                    ## IE5 Mac (2000)

                    Internet Explorer voor Mac, maar technisch eigenlijk compleet anders.

                    ---

                    ## Konqueror (2000)

                    Misschien wel belangrijker dan het klinkt.

                    Hieruit ontstond uiteindelijk WebKit → later gebruikt voor Safari.

                    ---

                    ## Firefox (2004)

                    Eigenlijk de goede opvolger van Netscape.

                    ---

                    ## Safari iOS (2007)

                    Andere browser dan Safari op Mac.

                    Belangrijk omdat het de eerste mobiele browser met touch events was.

                    (Nokia en Blackberry hadden daarvoor trouwens ook al browsers.)

                    ---

                    ## Chrome (2008)

                    Google werd gefrustreerd door Apple/WebKit omdat er weinig innovatie kwam.

                    Daarom hun eigen engine afgesplitst: Blink.

                    ---

                    ## Flow (2020)

                    Eigen rendering engine.

                    Maakte oorspronkelijk browsers voor smart tv’s.
                    Ondersteunt sommige CSS features niet.

                    ---

                    ## Ladybird (2024)

                    Begonnen als hobbyproject van iemand die bezig was af te kicken van drugs.

                    Best interessante browser-engine.

                    ---

                    ## Huidige belangrijkste browsers

                    * Chrome
                    * Safari
                    * Firefox

                    Alles wat ik maak moet eigenlijk minimaal op deze 3 werken.

                    ---

                    ## Igalia

                    Geen browsermaker, maar wel superbelangrijk in browserontwikkeling.

                    Open source bedrijf dat veel werkt aan implementaties van CSS en JavaScript in browsers.
