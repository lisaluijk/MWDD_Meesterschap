Mijn reflectie op de vakken van de minor

Sprint 0

Ik vind het concept van de website die ik voor sprint 0 had gemaakt erg leuk. De uitkomst viel me echter een beetje tegen. Ik merkte dat alles verder was weggezakt dan ik had gedacht. Ik vond het boekje wel leuk gelukt maar jammer dat het niet helemaal mijn eigen code was. Ik was echter wel erg trots op de illustraties die ik voor de website had gemaakt en de vibe die ze aan de site geven. Ik vind het leuk om dit soort uniekere websites te maken in plaats van een 1 uit een dozijn website. Mijn nieuwe inzichten vanuit sprint nul waren dat ik echt ff moest werken om mijn code kennis weer bij te spijkeren.

Browsertech

Ik vond browsertech een best leuk vak. De focus up ux vond ik erg interessant. Ook heb ik me veel bezig gehouden met progressive enhancement waardoor ik veel dingen in css heb opgelost in plaats van in javascript. Ook heb ik me een beetje bezig gehouden met het toegangkelijk maken van de website voor screenreaders wat ik ook wel interessant vond. Bij dit vak kwam ik op het einde in tijdsnood. Ik had me achteraf ingezien niet goed genoeg ingelicht op de randvoorwaarden en daarin ook niet de juiste prioriteit gelegd. Ik had nog graag wat dingen aangepast/afgemaakt zoals de validatie die nu niet overal helemaal soepel liep. Ook was ik al vrij ver met de vormgeving maar miste ik een paar laatste kleine dingetjes die het echt af konden maken.

CSS to the rescue

Ik vond css een erg leuke uitdaging. Hoewel ik wist dat het moeilijk zou worden heb ik ervoor gekozen om die opdracht wel te kiezen. Hij sprak me het meeste aan en ik dacht dat ik uit die opdracht ook de meeste energie zou kunnen halen. Ik vind het erg jammer dat het me niet was gelukt om hem binnen de tijd af te maken. Graag zou ik hem later nog afmaken en dit ga ik waarschijnlijk ook doen. Ik vind de approach die ik heb genomen op het idee van Nils ook erg leuk. Het was een grote uitdaging maar lag meer in de buurt van Javascript(wat ik denk ik het leukste vind om te coderen maar dat gaan we zien). Het was interessant om een soort JavaScript approach uit te werken in css. Ik heb hierdoor erg veel geleerd binnen css. Vooral de has en containerstyle queries heb ik helemaal uitgepluist. Ik denk ook zeker dat het mogelijk is 10 draaien te kunnen realiseren via deze manier dus ik hoop echt dat het me lukt dat nog voor het einde van de minor uit te werken en anders ga ik er zeker op een later moment nog naar kijken.

Hackaton

Tijdens de hackaton liep ik denk ik wel echt het meeste tegen mezelf aan. Ik keek er van te voren echt naar uit om me volledig op iets te storten en echt iets gaafs te maken. Uiteindelijk kwamen we erop uit dat ik een scrollbased animation ging maken. Ik had dit nog nooit gedaan en moest dit eerst leren. Dit was achteraf gezien niet een hele handige beslissing om dat in de hackaton zelf te doen. Uiteleindelijk was het me gelukt maar hadden we eigenlijk te weinig tijd om het echt uit te voeren. Daarna had ik meegeholpen met het idee om een blackhole te maken als je helemaal naar beneden scrollt. Joshua had het black hole gemaakt dus ik ging kijken naar de logica. Dit was met behulp van Chad gelukt. Helaas ging het niet goed met het koppelen van deze twee. Het mechanisme werkte maar toen we de code van Joshua erin probeerden te stoppen ging het kapot. Dit heb ik uiteindelijk samen met Arvid kunnen oplossen maar het was op het einde toch stuk gegaan en we hadden geen tijd meer om het te fixen. Ook had ik nog een 3d css sateliet gemaakt. deze vond ik eigenlijk wel goed gelukt maar hebben we uiteindelijk helaas ook niet gebruikt.

Human Centered Design

Ik vind het lastig een volledige review over dit vak te schrijven omdat ik het nog niet helemaal heb afgerond. Dus voor nu een kleine review en dan volgt er later een uitgebreidere review. Ik vond HCD een super interessant vak. Het was erg leuk om je volledig op 1 persoon te kunnen richten. Ik vond het ook super interessant om te luisteren naar de ervaringen van Darice en de dingen waar zij dagelijks tegenaanloopt. Het geeft me iets meer inzicht en ik kan me nu beter in haar verplaatsen. Ik merkte wel dat ik het lastig vond om me echt te richten op een prororype. Ik dacht namelijk vooral in oplossingen waarvan ik wist dat ze eventueel ook echt toegepast zouden kunnnen worden. Dit is ook een van mijn vuilkuilen geweest denk ik. Ik maakte het te groot en was te ambitieus voor de tijd die ik had met de skills die ik had. Ik vind het wel een super interessant vraagstuk. Ik hoop dat ik iets kleins maar fijns ervan ga kunnen maken.

API

Voor Api geld hetzelfde als voor browsertech. Ik heb het vak nu nog niet afgerond dus ik doe later eent weede reflectie als dat wel zo is. Ik vond api een heel interessant vak. Het was fijn om me alsnog te kunnen veridepen in de backend wat ik tijdens project tech niet had kunnen doen. Ik vond het ook erg leuk om meer in javascript te kunnen verdiepen. Ik vind ook dat ik een leuke visie heb voor de vormgeving alleen moet ik die nog goed zien uit te werken. Ik heb tijdens dit vak wel echt gerealiseerd dat je "backend" en "frontend" dus hoe het eruit ziet en hoe het werkt wel beter kan afwissleen met elkaar omdat 1 anders wel echt gaat achterlopen. Ik vond het ook erg leuk om javascript weer op te pakken en me ook verder te verdiepen ook al lag het tempo een beetje lager dan ik eigenlijk zou willen. Ik ben benieuwd naar wat ik er uiteindelijk van ga maken.
